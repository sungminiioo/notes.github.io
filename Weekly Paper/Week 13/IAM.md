# 🛡️ IAM은 무엇이며, 어떤 기능을 제공하나요?

## ⭐ IAM이란?

- *IAM(Identity and Access Management)**은

AWS에서 **사용자와 권한을 관리하는 서비스**로,

“누가(AI **Identity**) 무엇에(**Access**) 어떤 방식으로(**Management**) 접근할 수 있는지”를

세밀하게 제어하는 핵심 보안 서비스입니다.

쉽게 말해,

> AWS 리소스(EC2, S3, RDS 등)에 누가 어떤 행동을 할 수 있는지 결정하는 시스템
> 

입니다.

---

## 🔧 IAM이 제공하는 핵심 기능

### 1️⃣ **사용자(User) 관리**

AWS 계정의 실제 사람 또는 애플리케이션에 대한 “계정”을 생성할 수 있습니다.

각 사용자마다 권한을 따로 설정할 수 있어, 모든 사람이 root 계정을 쓸 필요가 없습니다.

---

### 2️⃣ **그룹(Group) 관리**

비슷한 역할을 가진 사용자들을 묶어

공통된 권한을 한 번에 부여할 수 있습니다.

예: `Admin`, `Developer`, `Viewer` 그룹 등

---

### 3️⃣ **역할(Role) 제공**

사용자뿐 아니라

**EC2, Lambda, ECS 같은 AWS 서비스 자체가 사용할 수 있는 권한**을 부여합니다.

예:

- EC2가 S3에서 파일을 읽기 위해 “S3ReadOnlyRole” 역할 부여
- Lambda 함수가 DynamoDB에 접근하는 Role 부여

---

### 4️⃣ **정책(Policy) 기반 권한 제어**

IAM 권한은 모두 JSON 형식의 Policy로 구성됩니다.

예:

- EC2 인스턴스 생성 허용
- S3 전체 접근 금지
- RDS 조회만 허용 등

정책을 통해 매우 정밀하게 권한을 설정할 수 있습니다.

---

### 5️⃣ **MFA(멀티팩터 인증)**

계정 보안을 강화하기 위해

비밀번호 + OTP(2차 인증)을 요구하는 기능입니다.

AWS 보안 베스트 프랙티스에서는

**root 계정에는 반드시 MFA 활성화**를 권고합니다.

---

### 6️⃣ **Access Key 관리**

AWS CLI, SDK 같은 프로그램이 AWS에 접근할 수 있도록

Access Key / Secret Key 방식의 인증을 제공합니다.

---

### 7️⃣ **외부 인증 연동 (SSO / Federation)**

기업용 시스템에서 사용하는

Google Workspace, Azure AD 같은 인증 시스템과 연동하여

SSO(Single Sign-On)를 구현할 수 있습니다.

---

# 📌 요약

| 구성 요소 | 설명 |
| --- | --- |
| User | AWS 리소스를 사용할 수 있는 사람/애플리케이션 |
| Group | 사용자 권한을 묶어 관리 |
| Role | AWS 서비스나 외부 서비스가 사용하는 권한 |
| Policy | JSON 기반 권한 규칙 |
| MFA | 계정 보안 강화 |
| Access Key | CLI/SDK 접근 키 |

# 🎯 결론

IAM은 AWS의 **보안과 권한 관리의 중심**이며,

**필요한 사용자에게 필요한 최소 권한만 부여하는 원칙(Least Privilege)**을 통해

안전한 클라우드 운영을 가능하게 해줍니다.

# 모범 답안

# 🛡️ 실습 해설 – IAM은 무엇인가요?

## ⭐ IAM이란 무엇인가요?

IAM(Identity and Access Management)은 AWS에서 제공하는 **접근 제어 및 권한 관리 서비스**입니다.

AWS 계정 내에서 **누가(Identity)** 어떤 리소스에 **얼마나(Access)** 접근할 수 있는지를 세밀하게 제어할 수 있으며,

AWS 환경을 안전하게 운영하기 위해 필수적으로 사용되는 기본 보안 구성 요소입니다.

간단히 말해,

> AWS 리소스 접근 권한을 관리하는 중앙 관리 시스템
> 

입니다.

---

## 🔧 IAM의 주요 기능

IAM은 AWS 리소스에 대한 권한을 체계적으로 관리하기 위한 다양한 기능을 제공합니다.

크게 4가지 핵심 요소로 구성됩니다.

### 1️⃣ 사용자(User)

- AWS 내부에서 개별 사람 또는 시스템을 “사용자”로 생성합니다.
- 각 사용자마다 개별적으로 권한을 부여할 수 있습니다.
- 개발자, 관리자, 서버 애플리케이션 등 다양하게 역할을 나눌 수 있습니다.

### 2️⃣ 그룹(Group)

- 여러 사용자를 하나의 그룹으로 묶어 공통된 권한을 부여할 수 있습니다.
- 예: `Developers`, `Admins`, `ViewOnly` 등
- 그룹에 권한을 부여 → 그룹에 속한 사용자들도 해당 권한을 자동 상속

### 3️⃣ 역할(Role)

- 특정 AWS 서비스 또는 외부 애플리케이션이 임시로 사용할 수 있는 권한입니다.
- 예:
    - EC2가 S3 버킷에 접근해야 할 때
    - Lambda가 DynamoDB에 접근해야 할 때
- 사용자 계정이 아닌 “서비스 단위”의 권한 부여 방식입니다.

### 4️⃣ 정책(Policy)

- JSON 형식으로 작성된 **권한 규칙 정의 파일**입니다.
- “어떤 리소스에 어떤 액션을 허용할 것인지”를 매우 세밀하게 정의할 수 있습니다.
- 이 정책을 사용자, 그룹, 역할(Role)에 연결해 권한을 부여합니다.

이를 통해 IAM은 **최소 권한 원칙(Least Privilege)**을 실현하게 해줍니다.

---

## 🔐 추가 보안 기능

IAM은 단순 권한 관리뿐 아니라 보안 강화를 위한 다양한 기능을 제공합니다.

### 🔸 MFA(Multi-Factor Authentication)

- 비밀번호 + OTP 같은 2차 인증을 요구하는 방식
- 계정 탈취 위험을 크게 줄여줌
- 특히 **Root 계정에는 필수**

### 🔸 조건부 접근(Condition)

- IP 주소, 시간대, MFA 여부 등 조건에 따라 접근을 허용/차단
- 예: 회사 네트워크에서만 접근 가능하도록 설정

---

## 🧩 실제 사용 예시

- 개발팀을 위한 IAM 그룹을 만들고,
    
    EC2와 S3의 읽기/쓰기 권한만 제공하며
    
    결제 관련 리소스는 접근 불가로 설정할 수 있습니다.
    
- 운영 중인 EC2 인스턴스가 S3 또는 DynamoDB 같은 AWS 서비스에 접근해야 할 경우,
    
    **IAM 역할(Role)**을 EC2에 연결하여 안전하고 자동화된 접근을 제공할 수 있습니다.

## 📌 요약

IAM은 AWS에서 가장 기본적이고 핵심적인 보안 서비스입니다.

| 구성 요소 | 역할 |
| --- | --- |
| 사용자(User) | 사람/시스템을 위한 계정 |
| 그룹(Group) | 여러 사용자에게 공통 권한 부여 |
| 역할(Role) | AWS 서비스/외부 앱이 사용하는 임시 권한 |
| 정책(Policy) | JSON 기반의 권한 규칙 |
