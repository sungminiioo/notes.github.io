# ⚙️ TypeScript의 동작 원리

---

## 🎯 TypeScript란?

TypeScript는 JavaScript에 **정적 타입(Type)** 개념을 추가한 프로그래밍 언어입니다.

하지만 브라우저는 TypeScript를 **직접 실행할 수 없습니다.**

그래서 TypeScript는 실행되기 전에 **JavaScript로 변환(컴파일)** 되어야 합니다.

---

# 🔍 TypeScript의 핵심 동작 과정

---

## 1️⃣ **Type Checking (타입 검사)**

TypeScript 컴파일러(`tsc`)는 먼저 **코드를 분석하며 타입 오류를 검사**합니다.

- 변수 타입이 맞는지 확인
- 함수의 매개변수/리턴 타입 확인
- 잘못된 접근, Null 가능성 등 검사

📌 이 과정은 **개발 중 오류를 빠르게 잡기 위해 존재**하며, 실행과는 무관합니다.

---

## 2️⃣ **Transpile → JavaScript 변환**

타입 검사 후, TypeScript 코드는 **브라우저가 이해할 수 있도록 JavaScript로 변환**됩니다.

- 타입 정보는 전부 제거됨
- 최신 문법도 설정에 따라 구버전 JS로 변환 가능 (ES6 → ES5)

➡️ 최종적으로 브라우저에 들어가는 것은 **순수 JavaScript**!

---

## 3️⃣ **JavaScript 실행**

변환된 JavaScript는 다음과 같은 환경에서 실행됩니다:

- 브라우저
- Node.js (백엔드)
- 서버리스 환경

⚠️ **TypeScript의 타입 정보는 실행 시점에는 존재하지 않음**

→ 오직 “개발 단계”에서만 도움을 줌.

---

# 🧠 TypeScript는 “정적 타입 검사 도구 + JS 변환기”이다!

다시 정리하면:

```
TS 코드 작성
   ↓ (타입 검사)
에러 있으면 알려줌
   ↓
TS → JS로 변환
   ↓
JS 실행
```

---

# 💡 TypeScript의 작동을 이해하는 핵심 포인트

### ✔ 브라우저는 TypeScript를 모른다

→ 반드시 JavaScript로 변환해야 한다.

### ✔ 타입 정보는 개발 단계에서만 사용된다

→ 컴파일 후엔 타입이 사라진다.

### ✔ 타입 검사는 “안전성”을 높이기 위한 과정

→ 실행 속도 개선이 목적이 아니다.

### ✔ tsc는 "타입 체크"와 "JS 변환"을 모두 수행하는 툴

→ Babel, SWC 등과 함께 사용 가능한 구조 (Next.js 등).

---

# 📌 결론

TypeScript는

**“타입 검사를 통해 코드를 더 안전하게 만들고, 실행을 위해 JS로 변환되는 언어”**입니다.

- 타입 검사 → 개발 생산성, 오류 감소
- JS 변환 → 실행 가능
- 런타임엔 타입이 없음

# ⚙️ TypeScript의 동작 원리

---

## 🎯 개요

타입스크립트는 자바스크립트의 상위 집합으로, 자바스크립트의 한계를 보완하며 더 안전하고 강력한 개발 경험을 제공합니다.

타입스크립트의 코드는 결국 자바스크립트로 변환되어 실행되지만, 그 과정에는 여러 중요한 단계가 포함되어 있어요.

---

# 1️⃣ 코드를 분석하여 구조화하기 (AST 생성)

타입스크립트 파일이 입력되면, 타입스크립트 컴파일러는 먼저 코드를 **추상 문법 트리(AST, Abstract Syntax Tree)** 로 변환합니다.

- AST는 코드의 구조를 트리 형태로 표현한 것
- 변수, 함수, 클래스 등 모든 코드 요소를 구조화
- 공백·주석 등 불필요한 요소 제거
- 컴파일러가 이해할 수 있는 형태로 먼저 정리하는 과정

---

# 2️⃣ 타입 검사로 안전성 확보하기

생성된 AST를 기반으로 **타입 검사(Type Checking)** 가 진행됩니다.

- 변수, 함수 매개변수, 반환 값 등 타입이 올바른지 검증
- 타입 충돌이 있으면 컴파일 단계에서 오류를 알려줌
- 예상치 못한 런타임 오류를 사전에 방지
- 코드 안정성을 높이고, 개발 생산성을 향상

예)

문자열을 받는 함수에 숫자를 전달하면 → 이 단계에서 즉시 오류 감지

---

# 3️⃣ 자바스크립트 코드로 변환하기 (Transpile)

타입 검사를 통과한 코드만 **자바스크립트(JavaScript)** 로 변환됩니다.

- 타입스크립트 전용 문법(타입, 인터페이스, 제네릭 등)은 모두 제거됨
- 순수 자바스크립트 코드만 남음
- 결과물은 브라우저나 Node.js에서 문제 없이 실행 가능하도록 변환됨

---

# 4️⃣ 자바스크립트 실행 과정

변환된 JavaScript 코드는 일반 JS와 동일하게 실행됩니다.

- JS 엔진이 다시 AST를 구성
- 바이트코드로 컴파일
- 이후 런타임에서 실행

➡️ 이 단계는 일반 자바스크립트와 동일한 실행 흐름입니다.

---

# 🛠 TypeScript를 사용하면 좋은 이유

타입스크립트의 변환 과정은 개발자가 **안정적**이고 **유지보수하기 쉬운 코드**를 작성하도록 돕습니다.

- 런타임 오류 감소
- 타입 오류를 "실행 전"에 발견
- 코드의 의도를 명확하게 표현 가능
- 협업 및 대규모 프로젝트에 특히 유리

📌 **정리하면:**

타입스크립트는 *자바스크립트로 변환되기 전에* 타입 안정성을 검사하여 더 안전하고 견고한 코드를 보장하는 언어입니다.
