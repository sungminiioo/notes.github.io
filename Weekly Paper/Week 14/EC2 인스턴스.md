# ⚙️ EC2 인스턴스

---

## 🎯 EC2란?

- *EC2(Amazon Elastic Compute Cloud)**는 AWS에서 제공하는 **클라우드 가상 서버** 서비스입니다.
- 쉽게 말해 **원하는 컴퓨팅 자원을 인터넷을 통해 바로 빌려 쓸 수 있는 서버**입니다.
- 서버를 직접 구매하고 관리할 필요 없이, **필요한 만큼만 사용**할 수 있습니다.
- “Elastic”이라는 이름처럼, 필요에 따라 **확장/축소(스케일링)**가 자유롭습니다.

---

# 🔍 EC2의 핵심 개념

---

## 1️⃣ **인스턴스 (Instance)**

- **정의:** EC2에서 실제로 실행되는 가상 서버 단위
- **특징:**
    - CPU, 메모리, 스토리지, 네트워크 등 리소스를 선택 가능
    - OS(예: Linux, Windows)를 선택하여 바로 사용 가능
- **활용:** 웹 서버, 애플리케이션 서버, 배치 작업 등 다양한 컴퓨팅 환경

---

## 2️⃣ **AMI (Amazon Machine Image)**

- **정의:** EC2 인스턴스를 만들기 위한 **서버 템플릿**
- **포함 내용:** 운영체제, 소프트웨어, 설정 등
- **활용:** 같은 환경의 서버를 반복해서 쉽게 생성 가능

---

## 3️⃣ **Key Pair (키 페어)**

- **정의:** 인스턴스 접속을 위한 **보안 인증 키**
- **특징:**
    - 공개키(Public Key)와 개인키(Private Key) 한 쌍
    - SSH 접속 시 개인키 사용
- **활용:** Linux 인스턴스 원격 접속

---

## 4️⃣ **Security Group (보안 그룹)**

- **정의:** 인스턴스로 들어오고 나가는 **트래픽을 제어하는 방화벽**
- **특징:** 포트, 프로토콜, IP 범위 설정 가능
- **활용:** HTTP/HTTPS 접근 허용, SSH 접근 제한 등

---

## 5️⃣ **Elastic IP**

- **정의:** EC2 인스턴스에 **고정 공인 IP를 부여**하는 기능
- **특징:** 인스턴스가 재시작되더라도 IP가 변하지 않음
- **활용:** 웹 서비스, 도메인 연결 등

---

# 🧠 EC2 사용 과정

```
1. AMI 선택 → OS/환경 결정
2. 인스턴스 타입 선택 → CPU/메모리/스토리지 선택
3. Key Pair 생성 → SSH 접속 준비
4. Security Group 설정 → 포트/방화벽 규칙 적용
5. 인스턴스 시작 → 서버 실행
6. Elastic IP 연결 (필요시) → 고정 IP 확보
7. 접속 및 애플리케이션 배포
```

# 💡 핵심 포인트

### ✔ 서버 구매 없이 바로 사용 가능

- 물리적 서버 관리 부담이 없음

### ✔ 확장성과 유연성이 뛰어남

- 필요에 따라 리소스를 늘리거나 줄일 수 있음

### ✔ 보안 관리 필수

- Security Group과 Key Pair를 적절히 설정해야 안전하게 운영 가능

### ✔ 다양한 활용 가능

- 웹/앱 서버, 데이터 처리, 배치 작업, 테스트 환경 등

---

# 📌 결론

**EC2는 AWS에서 제공하는 가상 서버 서비스로, 서버 구축과 운영을 쉽고 빠르게 할 수 있는 핵심 도구입니다.**

- AMI로 환경 구성 → 인스턴스로 실행 → Security Group으로 보호 → Key Pair로 접속
- 필요에 따라 확장/축소 가능 → 유연한 클라우드 인프라 제공

# 모범 답안

### EC2 인스턴스 생성 과정

EC2 인스턴스를 생성하는 과정은 AWS 콘솔을 통해 진행하는 것이 일반적입니다. 기본 흐름은 다음과 같습니다.

1. **인스턴스 시작 버튼을 클릭**합니다.
2. **사용할 운영체제 이미지(AMI)를 선택**합니다. 예를 들어 Amazon Linux, Ubuntu, Windows 등을 고를 수 있습니다.
3. **CPU와 메모리 사양을 선택하는 인스턴스 타입**을 고릅니다.
4. **키 페어를 생성하거나 기존 키를 선택**해 SSH 접속을 준비합니다.
5. **보안 그룹을 설정**해 인바운드, 아웃바운드 트래픽을 제한합니다.
6. 나머지 옵션을 검토 후 **인스턴스를 실행**하면 몇 분 내에 가상 서버가 동작합니다.
7. 필요에 따라 **EBS 볼륨 추가, 사용자 데이터 스크립트 등록** 같은 추가 설정도 가능합니다.

---

### EC2 인스턴스 관리 방법

생성된 인스턴스는 AWS 콘솔에서 쉽게 **시작, 중지, 재시작, 종료**할 수 있습니다.

- **AWS CLI**나 **SDK**를 이용해 명령어로 관리하거나 자동화도 가능
- **Auto Scaling 기능**을 사용하면 트래픽에 따라 자동으로 인스턴스 개수를 조정 가능
- **CloudWatch와 연동**해 모니터링과 알림을 설정할 수 있음

---

### EC2 인스턴스 보안 관리

EC2 인스턴스는 여러 단계의 **보안 기능**이 기본으로 제공됩니다.

1. **보안 그룹(Security Group)**
    - 인스턴스 단위의 가상 방화벽 역할
    - 어떤 포트를 열지, 어떤 IP에서 접속을 허용할지 설정 가능
2. **네트워크 ACL**
    - 보안 그룹보다 더 큰 단위인 **서브넷 수준**에서 트래픽 제어
3. **SSH 접속용 키 페어**
    - 개인 키를 이용해 안전하게 접속
4. **IAM Role**
    - 인스턴스가 AWS 리소스에 접근할 권한을 안전하게 위임 가능

➡ 이런 기능들을 통해 EC2는 **유연한 서버 운영**뿐 아니라 **기본적인 보안 체계**까지 함께 제공합니다.
