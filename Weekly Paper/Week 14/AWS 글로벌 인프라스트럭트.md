# ⚙️ AWS 글로벌 인프라스트럭처

---

## 🎯 AWS 글로벌 인프라스트럭처란?

AWS 글로벌 인프라는 **전 세계적으로 안정적이고 확장 가능한 클라우드 서비스를 제공하기 위해 설계된 물리적·논리적 구조**입니다.

주요 목표는:

- 고가용성(Availability)
- 지연 시간 최소화(Low Latency)
- 데이터 주권 및 규제 준수(Data Sovereignty)
- 글로벌 콘텐츠 배포(Content Delivery)

---

# 🔍 핵심 구성 요소

---

## 1️⃣ **Region (리전)**

- **정의:** 지리적으로 분리된 독립적 데이터 센터 그룹
- **특징:**
    - 최소 2개 이상의 가용 영역(AZ) 포함
    - 리전 선택으로 데이터 주권/규제 준수 가능
- **활용:** 특정 지역 사용자 대상 서비스 배포

---

## 2️⃣ **Availability Zone (AZ, 가용 영역)**

- **정의:** 리전 내 독립 데이터 센터 그룹
- **특징:**
    - 전력, 네트워크, 냉각 등 완전 독립
    - AZ 간 고속 저지연 연결 제공 (1~2ms)
    - 장애 발생 시 다른 AZ로 서비스 복구 가능
- **활용:** EC2, RDS 고가용성 배포

---

## 3️⃣ **Edge Location (엣지 로케이션)**

- **정의:** 콘텐츠 전송 및 캐싱(CDN)용 AWS 엔드포인트
- **특징:**
    - 사용자 근접 위치에서 빠른 콘텐츠 전달
    - Amazon CloudFront, Lambda@Edge 서비스 제공
- **활용:** 정적 웹사이트, 동영상 스트리밍

---

## 4️⃣ **Local Zone (로컬 존)**

- **정의:** 특정 대도시 근처에 위치한 AZ 확장
- **특징:**
    - 지연 시간 최소화, 일부 서비스만 제공
    - 기존 리전과 연결되어 고가용성 유지
- **활용:** 게임 서버, 미디어 처리, 초저지연 애플리케이션

---

## 5️⃣ **Wavelength Zone (웨이브렌스 존)**

- **정의:** 5G 네트워크 엣지에 배치된 AWS 인프라
- **특징:**
    - 모바일 네트워크와 직접 연결 가능
    - 초저지연 애플리케이션 지원 (AR/VR, IoT)
- **활용:** 실시간 모바일 서비스, IoT 처리

---

## 6️⃣ **Regional Edge Cache**

- **정의:** CloudFront의 콘텐츠 캐시 중간 계층
- **특징:**
    - 엣지 로케이션과 리전 사이에서 콘텐츠 캐싱
    - 리전 왕복을 줄여 응답 속도 개선
- **활용:** 글로벌 CDN 최적화

---

# 🧠 요약: AWS 글로벌 인프라 구조

```
Region (리전)
   ↓
Availability Zone (AZ) → 고가용성 배포
   ↓
Edge Location → 사용자 근접 CDN
Local Zone → 대도시 근접 초저지연
Wavelength Zone → 5G 엣지 초저지연
Regional Edge Cache → 캐시 중간 계층 최적화

```

---

# 💡 핵심 포인트

### ✔ 리전은 지리적 단위

- 데이터 주권, 규제, 지연 시간 고려

### ✔ AZ는 리전 내 독립 단위

- 장애 발생 시 빠른 서비스 복구 가능

### ✔ Edge Location과 Local Zone의 차이

- Edge Location → CDN용 캐시
- Local Zone → 실제 컴퓨팅 자원 제공, 초저지연

### ✔ Wavelength Zone은 5G 특화

- 모바일/IoT용 실시간 처리 가능

### ✔ Regional Edge Cache는 성능 최적화

- 엣지 → 리전 왕복 최소화

---

# 📌 결론

AWS 글로벌 인프라는 **고가용성 + 저지연 + 데이터 주권 + 글로벌 콘텐츠 배포**를 위해 설계된 체계적 구조입니다.

- 리전 & AZ → 안정적 서비스 배포
- Edge Location & Regional Edge Cache → 글로벌 CDN
- Local Zone & Wavelength Zone → 초저지연 애플리케이션 지원

# 모범 답안

---

### 리전 (Region)

AWS에서 말하는 리전은 쉽게 말해 **전 세계에 흩어져 있는 AWS의 큰 거점**입니다.

지역별로 물리적으로 나뉜 인프라라고 보면 되고요. **하나의 리전 안에는 여러 개의 데이터 센터가 묶여서 돌아갑니다.**

- 특징: **리전끼리는 서로 완전히 독립적**
- 예시: 서울 리전(`ap-northeast-2`), 도쿄 리전(`ap-northeast-1`)
- 의미: 서울 리전에 장애가 나더라도 도쿄 리전 서비스는 멀쩡히 돌아감

---

### 가용 영역 (Availability Zone, AZ)

리전 안으로 들어가 보면 또 쪼개져 있습니다. 바로 **가용 영역(AZ)**인데요, 각각 **물리적으로 분리된 데이터 센터 클러스터**라고 생각하면 됩니다.

- 특징:
    - 전력, 냉각, 네트워크 인프라가 각각 독립
    - 한 AZ가 망가져도 리전 전체가 멈추지 않음
    - 리전 안 AZ끼리는 **초저지연 네트워크**로 연결 → 빠른 통신 가능
- 구성: AWS에서는 최소 2개, 많게는 3~4개 AZ가 리전마다 존재

---

### 엣지 로케이션 (Edge Location)

마지막으로 **엣지 로케이션**이 있습니다.

- 특징:
    - 리전이나 AZ보다 훨씬 더 많은 수로 전 세계 곳곳에 분포
    - 사용자가 접속하는 **가장 가까운 위치에서 요청 처리**
- 활용: 주로 **CloudFront 같은 CDN 서비스**가 엣지 로케이션 기반
- 장점: 이미지나 정적 리소스 같은 콘텐츠를 **더 빠르고 지연 없이 전달**, 사용자 경험 개선

---

### 구성 요소 연결 구조

이 세 가지는 **계층처럼 유기적으로 돌아갑니다**.

1. **리전**: 가장 큰 단위 → 물리적 지역 분산 담당
2. **AZ**: 장애 복원력과 고가용성 확보
3. **엣지 로케이션**: 사용자에게 빠른 응답성 제공

➡ 이렇게 나눠져 있어서 AWS 인프라는 **장애에도 끄떡없고, 속도도 빠르며, 글로벌 서비스 확장도 수월**합니다.
