## 📝 핵심 개념 요약

- **IP 주소**는 컴퓨터의 주소이며, 퍼블릭(인터넷용) / 프라이빗(VPC 내부용)으로 구분됨
- **VPC**는 AWS 내부의 독립된 네트워크 공간
- **CIDR**은 IP 범위를 표현하는 방식 (예: /24, /16)
- **서브넷**은 VPC를 기능/보안 용도별로 분리한 네트워크 조각
- **퍼블릭 서브넷**은 인터넷과 통신 가능, **프라이빗 서브넷**은 직접 통신 불가
- **IGW**는 VPC가 인터넷과 연결되도록 하는 관문
- **NAT Gateway**는 프라이빗 서브넷 → 인터넷(아웃바운드)만 가능하도록 도와줌
- **기본 VPC는 학습용**, **사용자 정의 VPC는 실무용**

- IP 구조 및 서브넷 구성 이해
- 기본 VPC · 사용자 정의 VPC 구조 비교
- Public Subnet / Private Subnet 구성 흐름 학습
- IGW 연결 흐름 확인
- NAT Gateway의 아웃바운드-only 원리 학습

### ❗ 헷갈렸던 점 / 문제 해결:

### ❓ 1. Public IP와 Private IP의 차이를 헷갈렸음

→ 비유(아파트 주소 vs 동/호수)로 완전히 해결됨

### ❓ 2. NAT Gateway가 들어오고 나가는 역할 모두 하는 줄 알았음

→ 정답: **나가는 요청만 지원(outbound only)**

→ 인터넷 → 프라이빗 서버 직접 접근은 불가능

### ❓ 3. Subnet이 가용영역(AZ)와 어떤 관계인지 헷갈렸음

→ 한 Subnet = 반드시 한 AZ에 종속

→ 고가용성을 위해 여러 AZ에 분산해야 함

### 💡 느낀 점 / 배운 점:

- AWS 네트워킹은 실생활 공간(아파트, 보안, 출입문) 비유로 이해하면 매우 쉬워짐
- EC2가 여러 IP를 갖는 이유(Public + Private)를 명확히 이해하게 됨
- 실무에서는 기본 VPC를 거의 사용하지 않고, 반드시 Custom VPC로 구성해야 한다는 이유(보안/구조 제어)를 알게 되었다
- NAT, IGW 등 개념만 알고 있었지만, 내부/외부 흐름을 단계별로 이해하면서 네트워크 흐름 전체가 눈에 보이기 시작함
