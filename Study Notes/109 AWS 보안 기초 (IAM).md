## 📝 핵심 개념 요약

### 1. IAM 개념

- **IAM = Identity and Access Management** → 사용자와 권한을 관리하는 시스템
- **비유:**
    - 회사 건물 = AWS
    - 직원 = IAM 사용자
    - 층/부서 = 서비스(EC2, S3 등)
    - 출입카드 = IAM 정책
    - 보안팀 = IAM 시스템

---

### 2. IAM 핵심 원칙

1. **최소 권한 원칙(Least Privilege)**
    - 필요한 권한만 부여
    - 예: 인턴은 특정 사무실만 출입 가능
2. **역할 기반 접근(Role-Based Access Control, RBAC)**
    - 직무/그룹에 따라 권한 관리
    - 예: 개발팀 → EC2/RDS, 디자인팀 → S3

---

### 3. IAM 구성 요소

| 구성요소 | 비유 | 설명 |
| --- | --- | --- |
| 사용자(User) | 개별 직원 | AWS 콘솔 또는 프로그래밍 접근용, 고정된 권한 |
| 그룹(Group) | 부서/팀 | 비슷한 업무 사용자 묶어 권한 관리, 효율적 |
| 역할(Role) | 임시 출입증 | 임시 권한 부여, 서비스 간 접근, 외부 협업 시 사용 |
| 정책(Policy) | 출입카드 규칙서 | JSON 형태로 “누가, 무엇을, 어디서, 언제, 허용/거부” 정의 |

---

### 4. IAM 정책 종류

1. **AWS 관리형 정책** → 기본 템플릿, 즉시 사용 가능
2. **고객 관리형 정책** → 맞춤형 권한 생성, 직접 관리
3. **인라인 정책** → 특정 사용자에게만 1회성 부여, 공유 불가

---

### 5. 루트 사용자 vs IAM 사용자

- **루트 사용자**
    - 모든 권한 보유, 계정 삭제 가능
    - 일상 작업 금지, MFA 필수
- **IAM 사용자**
    - 특정 권한만 보유
    - 루트 사용자를 대신해 일상 운영 가능
    - 권한 최소화로 보안 강화

## 📊 핵심 요약 표

| 항목 | 설명 |
| --- | --- |
| IAM | AWS 사용자 및 권한 관리 시스템 |
| 핵심 원칙 | 최소 권한, 역할 기반 접근 |
| 구성요소 | User, Group, Role, Policy |
| 정책 종류 | AWS 관리형, 고객 관리형, 인라인 |
| 루트 vs IAM | 루트 = 마스터키, IAM = 최소 권한 운영 |

### 💻 실습 내용 정리

- IAM 사용자 생성
    - 사용자 지정 암호 설정, 그룹(admin) 생성 후 권한 부여
    - IAMFullAccess, AmazonEC2FullAccess, EC2InstanceConnect, AmazonS3FullAccess, AmazonRDSFullAccess 추가
- IAM 사용자로 로그인 확인
- IAM 역할을 이용한 EC2-S3 접근 실습 계획
- 루트 사용자 최소 사용 원칙 적용

### ❗ 헷갈렸던 점 / 문제 해결:

| 문제 | 해결 |
| --- | --- |
| 루트 사용자와 IAM 사용자 혼동 | 루트 = 계정 소유자, IAM = 일반 운영용 |
| EC2InstanceConnect와 EC2FullAccess 차이 | EC2InstanceConnect = 브라우저 SSH 접속, EC2FullAccess = 리소스 관리 전체 |
| 정책 관리 효율성 | 그룹 및 고객 관리형 정책 사용 권장, 인라인 정책 최소 사용 |

### 💡 느낀 점 / 배운 점:

- 루트 사용자 사용 최소화 → 단일 실패 지점 방지
- IAM 사용자, 그룹, 역할 활용 → 권한 최소화와 효율적 관리
- 정책 구조 이해 → JSON 정책 직접 읽고 작성 가능
- 서비스 간 안전한 접근(EC2 → S3) → 역할(Role) 활용
