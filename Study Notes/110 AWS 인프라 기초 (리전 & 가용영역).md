## 📝 핵심 개념 요약

### 1. 리전(Region)

- **정의:** AWS 데이터센터가 위치한 물리적 지역
- **역할:** 전 세계 서비스를 분산 배포할 수 있는 단위
- **선택 기준:**
    1. **네트워크 전송 속도** → 사용자와 가까울수록 빠름
        - 예: 한국 서비스 → 서울 리전, 미국 서비스 → 미국 리전
    2. **법적/규제 요건** → 금융, 전자금융업, PG사 등은 국내 리전 필수
    3. **리전별 서비스 제공 차이** → 모든 서비스가 모든 리전에 있는 것은 아님
    4. **비용** → 리전마다 서비스 요금 상이

---

### 2. 가용영역(AZ, Availability Zone)

- **정의:** 리전 내 여러 데이터센터를 묶은 그룹
- **특징:**
    - 리전당 보통 3개 이상의 AZ 구성
    - 각 AZ는 2~6개의 데이터센터 그룹
    - AZ 간 거리는 약 100km 이내로 유지 (장애 격리 목적)
- **역할:** 장애 발생 시 다른 AZ로 서비스 이전 가능, 고가용성 확보

## 📊 핵심 요약 표

| 구분 | 설명 | 특징 |
| --- | --- | --- |
| 리전(Region) | AWS 데이터센터 위치 | 사용자와 가까울수록 속도 빠름, 법적 규제 준수, 비용·서비스 차이 |
| 가용영역(AZ) | 리전 내 데이터센터 그룹 | 장애 격리, 고가용성 확보, 2~6개 데이터센터 그룹, 리전 내 3개 이상 |

### 💻 실습 내용 정리

- **서울 리전 선택** 후 EC2, S3 생성 실습
- **AZ 선택**: EC2 인스턴스를 여러 AZ에 분산 배치
- 리전별 서비스 제공 여부 확인 → 특정 리전에서만 제공되는 서비스 테스트
- 비용 비교: 동일 서비스 다른 리전에서 가격 차이 확인

### ❗ 헷갈렸던 점 / 문제 해결:

| 문제 | 해결 |
| --- | --- |
| 리전과 AZ 개념 혼동 | 리전 = 물리적 지역, AZ = 리전 내 데이터센터 그룹 |
| AZ 간 거리와 장애 격리 목적 | AZ 간 100km 내외, 장애 발생 시 다른 AZ로 서비스 전환 |
| 서비스 제공 차이 | 리전별 제공 서비스 확인 필수 |

### 💡 느낀 점 / 배운 점:

- 리전 선택은 속도, 법적 요구, 비용, 서비스 차이를 모두 고려해야 함
- AZ 개념 이해 → 고가용성 설계 및 장애 대비 가능
- 글로벌 서비스를 설계할 때, 리전과 AZ 배치 전략이 중요
