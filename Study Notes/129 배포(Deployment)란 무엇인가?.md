> 

---

## 📝 핵심 개념 요약

- **배포란** 개발이 끝난 웹사이트/애플리케이션을 **실제 사용자들이 접근할 수 있도록 서버에 올리는 과정**
- 로컬 PC에서만 실행되던 프로그램을 **인터넷에 공개**하는 단계
- 개발환경과 운영환경은 목적·보안·성능·구성이 모두 다르기 때문에 반드시 분리해서 관리해야 함
- 배포는 단순 업로드가 아니라 **안정성, 보안, 성능을 고려한 준비 과정**

## 📊 핵심 요약 표

| 구분 | 개발환경 (Development) | 운영환경 (Production) |
| --- | --- | --- |
| 실행 위치 | 내 PC / 노트북 | 실제 서버 (클라우드) |
| 접속 주소 | localhost:3000 | [www.my-site.com](http://www.my-site.com/) |
| 접근 가능자 | 나 혼자 | 전 세계 사용자 |
| 목적 | 개발·실험·테스트 | 실제 서비스 제공 |
| 안정성 요구 | 낮음 | 매우 높음 |
| 보안 수준 | 느슨함 | 엄격함 |

### ❗ 헷갈렸던 점 / 문제 해결:

### ❓ 헷갈렸던 점

- “그냥 깃허브에 올리면 배포 아닌가?”라는 오해
- 로컬에서는 잘 되는데, 서버에서는 안 되는 이유가 명확하지 않았음

### ✅ 이해한 내용

- 깃허브 업로드 ≠ 배포
- **배포는 서버 + 네트워크 + 보안 + 운영 환경 설정까지 포함**
- 포트(3000 vs 80/443), DB, 보안 설정 차이가 핵심 원인

### 💡 느낀 점 / 배운 점:

- 배포는 단순한 마지막 단계가 아니라 **서비스의 시작 단계**
- 개발환경과 운영환경을 분리하지 않으면 사고 나기 쉬움
- “내 컴퓨터에서만 잘 되는 코드”는 실제 서비스와는 전혀 다를 수 있음
- 이후 AWS 배포 실습을 이해하기 위한 아주 중요한 개념이라는 걸 느낌
