## 📝 핵심 개념 요약 

- **TypeScript = 자바스크립트 + 타입 시스템**
- JS는 **동적 타이핑**, TS는 **정적 타이핑** → 타입 오류를 사전에 방지
- 컴파일 단계에서 오류를 잡아 **런타임 에러 감소**
- 대규모 프로젝트에서 **코드 안정성, 유지보수성, 생산성 향상**

## 📊 핵심 요약 표

| 개념 | 설명 |
| --- | --- |
| TypeScript | JS의 상위 문법(Superset). 타입 검사 기능 추가 |
| 동적 타이핑 | 실행 중 타입 결정 (JS) → 오류 발견 늦음 |
| 정적 타이핑 | 컴파일 시 타입 체크 (TS) → 오류 빠르게 발견 |
| 장점 | 안정성↑, 자동완성↑, 협업/유지보수 용이 |

### 💻 실습 내용 정리

### 🔍 동적 타입에서 발생하는 문제 예시

```tsx
function calculateTotal(price, tax) {
  return price + tax;
}

const result1 = calculateTotal("100", "10");  // "10010"
const result2 = calculateTotal(100);          // NaN
const result3 = calculateTotal(100, "10%");   // "10010%"
```

### 👉 문제점

- 문자열로 연결되거나
- `undefined + number`로 NaN 발생
- 실수 발생해도 **런타임 전에는 알 수 없음**

### ❗ 헷갈렸던 점 / 문제 해결:

### 헷갈렸던 점

- JS에서 타입이 왜 문제를 일으키는지 명확히 체감이 부족함

### 문제 해결

- 타입스크립트가 **컴파일 단계에서 오류를 미리 알려준다는 점**을 이해
- 타입 명시 → 실수한 코드가 바로 에러로 표시됨
- “동적 타입”과 “정적 타입”의 차이를 예제 코드로 직접 확인하면서 완전히 이해됨

### 💡 느낀 점 / 배운 점:

- 타입스크립트는 단순히 “코드에 타입을 다는 언어"가 아니라
    
    **프로젝트 규모가 커질수록 개발 효율과 안정성을 확 높여주는 도구**라는 걸 실감함
    
- 특히 자동완성, 에러 사전 차단 기능이 JS와 큰 차이를 만듦
- 앞으로 서버·프론트 모두 TS로 리팩토링하고 싶다는 생각이 들어 큰 동기부여 됨
