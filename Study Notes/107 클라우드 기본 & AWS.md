## 📝 핵심 개념 요약

### 1. 클라우드 컴퓨팅

- **클라우드**: 필요할 때 쉽게 사용하고 반납 가능한 컴퓨터 자원 (서버, 저장소, DB 등)
- **클라우드 컴퓨팅**: 클라우드 자원에 어디서나 접근 가능하게 하는 기술
- **클라우드 특징 5가지**
    1. 온디맨드 셀프 서비스
    2. 광범위한 네트워크 접근
    3. 리소스 풀링
    4. 신속한 탄력성
    5. 측정된 서비스

### 2. 온프레미스 vs 클라우드

| 구분 | 온프레미스 | 클라우드 |
| --- | --- | --- |
| 초기 비용 | 서버 직접 구매 수천만~억 | 거의 없음 |
| 관리 | IT 담당자가 직접 | 클라우드 업체 관리 |
| 확장 | 서버 주문 → 설치 (몇 주~월) | 클릭 몇 번 (몇 분) |
| 위험 | 하드웨어 고장 시 중단 | 클라우드 안정성 보장 |

### 3. 클라우드 컴퓨팅 장점

- **비즈니스 민첩성**: 서버 준비 시간 단축, 빠른 테스트 및 출시
- **비용 절감**: 초기 투자 제로, 사용한 만큼만 지불, 인력 절약
- **운영 효율성**: 핵심 업무 집중, 자동화, 전문성 활용
- **글로벌 서비스**: 전 세계 배포, 빠른 속도

### 4. 클라우드 서비스 모델

| 모델 | 제공 | 개발자 역할 | 예시 | 비유 |
| --- | --- | --- | --- | --- |
| IaaS | 가상 서버, 저장소, 네트워크 | OS/앱 설치 및 배포 | EC2, GCE | 빈 사무실(책상/의자 직접 준비) |
| PaaS | 개발 플랫폼(운영체제+도구) | 코드 작성 및 배포 | Elastic Beanstalk, Heroku, Vercel | 완비된 사무실(책상/컴퓨터 준비) |
| SaaS | 완성된 애플리케이션 | 사용만 함 | Gmail, Notion, Figma | 호텔(모든 준비 완료) |

### 5. 클라우드 배포 모델

| 모델 | 설명 | 장점 | 단점 | 예시 |
| --- | --- | --- | --- | --- |
| 퍼블릭 클라우드 | 누구나 인터넷 사용 가능 | 저렴, 관리 편함, 빠른 확장 | 보안 우려 | AWS, GCP, Azure |
| 프라이빗 클라우드 | 특정 기업만 사용 | 높은 보안, 통제 | 비쌈, 관리 복잡 | 삼성전자, 국민은행 |
| 하이브리드 | 퍼블릭 + 프라이빗 혼합 | 유연성, 비용·보안 균형 | 관리 복잡 | 현대차, 정부기관 |

### 6. AWS 소개

- **Amazon Web Services**: 아마존이 제공하는 클라우드 컴퓨팅 서비스 (설립 2006, 시장 점유율 1위)
- **배경**: 아마존 내부 인프라 효율화 → 남는 서버 활용 → 외부에 클라우드 서비스 제공
- **주요 서비스**
    - 컴퓨팅: EC2
    - 저장소: S3
    - DB: RDS
    - AI/ML, 네트워크, 보안 등 200+ 서비스
- **인프라 구조**
    - 리전(Region): 전 세계 주요 도시 데이터센터
    - 가용 영역(AZ): 리전 내 독립 데이터센터
    - 엣지 로케이션: CDN, DNS 캐시 서버
- **결제 모델**: Pay-as-you-go, 사용한 만큼 지불, 무료 체험 가능
