## 📝 핵심 개념 요약

### 1. 인스턴스 중지 vs 종료

| 구분 | 의미 | 특징 | IP 변화 |
| --- | --- | --- | --- |
| **중지(Stop)** | 인스턴스 동작만 멈춤 | 내용(디스크, 설정) 그대로 보존, 언제든 재시작 가능 | Public IP 변경 가능 |
| **종료(Terminate)** | 인스턴스 삭제 | 영구 삭제, 복구 불가 | N/A |

💡 인스턴스를 중지하면 재시작 시 Public IP가 바뀌므로, 도메인/방화벽/클라이언트 설정에 영향을 줄 수 있음

---

### 2. Elastic IP (EIP)

- **정의:** AWS에서 고정 Public IP를 제공하는 서비스
- **용도:** 인스턴스 중지/재시작 시에도 Public IP 유지 → 서비스 안정성 확보
- **비용:** 프리티어 무료 아님, 미사용 시 $3.6/월

---

### 3. 관리 단계

1. **Elastic IP 생성**
    - 네트워크 및 보안 → 탄력적 IP → 생성 후 체크 → 작업 클릭
2. **인스턴스 연결**
    - 생성한 Elastic IP 선택 → 연결 실행
3. **중지 후 재시작 시 고정 IP 유지 확인**

### 💡 느낀 점 / 배운 점:

- 중지와 종료의 차이를 확실히 이해 → 불필요한 비용 방지
- 실 서비스에서는 **Elastic IP 필수** → 도메인 연결 및 방화벽 문제 예방
- AWS 네트워크 설정은 서비스 안정성과 직결됨
