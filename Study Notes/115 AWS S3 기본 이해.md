## 📝 핵심 개념 요약

### 1. S3 정의

| 개념 | 설명 |
| --- | --- |
| S3 | 클라우드 저장소 서비스, 어디서나 접근 가능, 사용한 만큼 비용 지불 |
| 비유 | 무제한 클라우드 하드디스크, Google Drive/Dropbox 유사하지만 개발자용 |

### 2. 핵심 구성 요소

| 구성요소 | 설명 |
| --- | --- |
| **버킷(Bucket)** | 최상위 저장소 컨테이너, 전 세계에서 유일한 이름 필요 |
| **객체(Object)** | 실제 저장 파일 + 메타데이터 + 고유 키 |
| **키(Key)** | 객체의 전체 경로, 파일 주소 역할 (`폴더/파일명`) |

### 3. S3 vs 다른 저장소

| 저장소 | 장점 | 단점 |
| --- | --- | --- |
| 로컬 하드디스크 | 빠름, 무료 | PC 내 접근만 가능, 고장 시 데이터 손실 |
| Google Drive/Dropbox | 어디서나 접근, 쉬움 | 개발용 기능 부족, 용량 제한 |
| AWS S3 | 무제한, 높은 내구성, 전 세계 접근, 개발자 친화 | 설정 복잡, 비용 발생 |

### 4. S3 사용 사례

- 웹사이트 호스팅 (정적 웹사이트, React/Vue 빌드 결과물)
- 파일 저장소 (사용자 업로드 이미지, 동영상)
- 백업/아카이브 (DB 백업, 로그 파일)
- 빅데이터 저장 (머신러닝 학습 데이터, IoT 센서 데이터)

## 📊 핵심 요약 표

| 개념 | 설명 |
| --- | --- |
| 버킷 | S3 저장소 컨테이너, 유일한 이름 필요 |
| 객체 | 파일 데이터 + 메타데이터 + 고유 키 |
| 키 | 객체 경로, S3 내 주소 역할 |
| 퍼블릭 액세스 | 기본 차단, 필요 시 웹사이트 공개 설정 가능 |
| 정적 웹사이트 호스팅 | S3만으로 HTML/CSS/JS 사이트 운영 가능 |
| 스토리지 클래스 | Standard(자주 접근), Intelligent-Tiering(자동 최적화), Glacier(장기 보관) |

### 💻 실습 내용 정리

| 단계 | 내용 |
| --- | --- |
| 1 | AWS 콘솔에서 S3 접속 → 버킷 생성 (서울 리전, 퍼블릭 액세스 차단 유지) |
| 2 | 파일 업로드 → 키를 이용한 폴더 구조 생성 |
| 3 | 업로드된 파일 URL 확인 및 접근 테스트 |
| 4 | 웹사이트 호스팅 활성화 → index.html, error.html 설정 |
| 5 | Node.js/React/Express에서 S3 파일 업로드 및 사용 실습 |

### ❗ 헷갈렸던 점 / 문제 해결:

| 문제 | 해결 |
| --- | --- |
| S3에 업로드한 파일 기본적으로 접근 불가 | 퍼블릭 액세스 허용 또는 버킷 정책 설정 |
| 버킷 이름 중복/규칙 위반 | AWS 요구사항에 맞춰 이름 지정 (`소문자, 하이픈, 전 세계 유일`) |

### 💡 느낀 점 / 배운 점:

- S3는 단순 저장소이지만, 개발 환경에서 웹 호스팅, 백업, 빅데이터 저장 등 폭넓게 활용 가능
- 권한 관리 중요 → 실수 시 민감 데이터 노출 위험
- 정적 웹사이트 호스팅으로 별도 서버 없이도 사이트 운영 가능
