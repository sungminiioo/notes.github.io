## 📝 핵심 개념 요약

- AWS는 **사용량 기반 과금** 구조 → 실수로 비용 발생 가능
- 프리티어 정책 변경으로 인해 **예산 알림 설정은 필수**
- 예산(Budget)은 **비용을 자동으로 막아주지 않음**
    - → 비용 발생 시 **알림만 제공**
- 최소 2가지 예산 설정 권장
    1. **제로 지출 예산**: 프리티어 초과 즉시 감지
    2. **월별 비용 예산**: 전체 비용 한도 관리
- 알림을 받으면 **즉시 리소스 점검 및 정리** 필요

## 📊 핵심 요약 표

| 예산 종류 | 금액 | 목적 | 알림 조건 |
| --- | --- | --- | --- |
| 제로 지출 예산 | $0.01 | 프리티어 초과 감지 | 실제 지출 초과 시 |
| 월별 비용 예산 | $25 | 월 전체 비용 관리 | 85%, 100%, 예상 100% |

### 💻 실습 내용 정리

## 1️⃣ 예산 1: 제로 지출 예산 (필수)

```
예산 템플릿: 제로 지출 예산
목적: 프리티어 초과 즉시 감지
알림 조건: 실제 지출이 0.01 USD 초과 시
```

- 프리티어 범위를 **조금이라도 벗어나면 즉시 알림**
- 가장 중요한 안전장치

---

## 2️⃣ 예산 2: 월별 비용 예산 (권장)

```
예산 템플릿: 월별 비용 예산
예산 금액: $25
목적: 월간 전체 지출 한도 관리
알림 조건:
1) 실제 지출85% 도달
2) 실제 지출100% 도달
3) 예상 지출이100%에 도달할 것으로 예측될 때
```

- 비용 폭주 사전 차단용
- “이미 초과” + “곧 초과” 상황 모두 감지 가능

---

## ⚠️ 주의사항

- 예산 알림은 **자동 리소스 종료 기능이 아님**
- 알림 수신 후 반드시 **사람이 직접 조치**해야 함
- 프리티어 정책
    - 예산 **2개까지 무료**
    - 3개부터 월 **$0.60 추가 비용 발생**
- 실습 종료 후
    - EC2 인스턴스 중지
    - 사용하지 않는 RDS 삭제
    - 불필요한 리소스 정리 필수

---

## 🚨 예산 초과 시 대응 절차

1. AWS 콘솔 로그인 → **Billing Dashboard 확인**
2. 좌측 메뉴에서
    - `청구서`
    - `프리 티어`
        
        메뉴 확인
        
3. 어떤 서비스에서 비용이 발생했는지 분석
4. **불필요한 리소스 즉시 정리**
5. 팀장 또는 강사에게 상황 공유

### ❗ 헷갈렸던 점 / 문제 해결:

- **예산 설정하면 비용이 자동으로 멈춘다?**
    - ❌ 아님
    - ⭕ 알림만 제공, 조치는 수동
- **프리티어면 예산 필요 없다?**
    - ❌ 프리티어 초과는 언제든 발생 가능
- **예산 개수 많을수록 좋은가?**
    - ❌ 3개부터 비용 발생

### 💡 느낀 점 / 배운 점:

- AWS 비용 관리는 **설정하지 않으면 사고로 이어질 수 있음**
- 예산 알림은 선택이 아니라 **기본 보안 설정**
- “설정해두면 안심”이 아니라
    
    **알림 후 즉시 행동하는 습관**이 중요함
    
- 클라우드는 편리한 만큼 **책임도 사용자에게 있음**을 체감함
