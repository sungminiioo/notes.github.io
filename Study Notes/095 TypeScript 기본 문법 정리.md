## 📝 핵심 개념 요약

- TS는 런타임이 아니라 정적 타입 시스템 → 컴파일(TSC) 단계에서 타입 검사
- 원시 타입(string/number/boolean/null/undefined)과 any/unknown의 차이
- 리터럴 타입, 유니온/인터섹션 타입
- 배열 타입, 튜플 타입(“길이와 순서가 고정된 배열”)
- 객체 타입(type alias/interface), 옵션 속성, 중첩 객체, 인덱스 시그니처
- 함수 타입: 매개변수, 반환 타입, void/never, async 함수 타입
- 실습: 기본 블로그 타입 정의

## 📊 핵심 요약 표

| 개념 | 설명 |
| --- | --- |
| 기본 타입 | string, number, boolean, null, undefined |
| any / unknown | any = 모든 타입 허용 / unknown = 사용 전 타입 체크 필요 |
| 리터럴 타입 | 특정 값만 허용하는 타입 |
| 유니온/인터섹션 | 여러 타입 중 하나 / 모든 조건 충족 |
| 배열 타입 | number[], string[] 등 |
| 튜플 | 요소 개수·순서가 고정된 배열 |
| 객체 타입 | type alias, interface, 옵셔널 속성, 중첩 타입 |
| 함수 타입 | 매개변수 타입 필수, return 생략 가능(추론됨) |
| never | 절대 반환하지 않는 함수 |

### ❗ 헷갈렸던 점 / 문제 해결:

### 헷갈렸던 점

- `unknown`과 `any`의 차이를 명확히 이해하기 어려웠음
- 튜플과 일반 배열의 차이가 헷갈렸음
- 함수 타입 지정에서 “return 생략하면 추론되는데 굳이 타입 지정해야 하나?” 고민됨

### 문제 해결

- unknown → 반드시 타입 체크 후 사용해야 한다는 점으로 명확히 구분
- 튜플은 “길이와 위치가 고정된 배열”이라는 개념으로 정리
- 함수는 실무 컨벤션상 return 타입 명시를 권장한다는 기준으로 정리

### 💡 느낀 점 / 배운 점:

- 타입스크립트는 “런타임이 아니라 개발 중 안전성”을 확보하기 위한 언어라는 점이 더 명확해짐
- 타입을 상세하게 정의하는 습관이 구조적인 프로젝트에서 매우 중요함
- 튜플·인터섹션 등 기존 JS에서 없던 개념들이 로직을 더 명확하게 설계하게 해줌
- 함수 타입 정의가 많아 보이지만, 오히려 코드 가독성을 높여 줌
