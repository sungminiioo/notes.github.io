## 📝 핵심 개념 요약

### 1. EC2(Elastic Compute Cloud)

- **정의:** 클라우드 상의 가상 컴퓨터
- **특징:**
    - 필요할 때 생성/삭제 가능
    - 사용한 만큼 요금 지불
    - 범용적 서버 용도로 사용 가능

### 2. 인스턴스(Instance)

- 하나의 가상 서버
- 독립적인 OS와 리소스 보유

### 3. AMI (Amazon Machine Image)

- 인스턴스 초기 설정
- OS, 설치된 소프트웨어, 기본 설정 포함

### 4. 인스턴스 타입

| 타입 | vCPU | 메모리 | 용도 |
| --- | --- | --- | --- |
| t3.nano | 2 | 0.5 GB | 개발/테스트 |
| t3.micro | 2 | 1 GB | 소규모 웹사이트 |
| t3.small | 2 | 2 GB | 경량 웹서버 |
| t3.medium | 2 | 4 GB | 중소규모 웹앱 |
| t3.large | 2 | 8 GB | 웹 애플리케이션 |
| t3.xlarge | 4 | 16 GB | 중간 규모 앱 |

## 💻 EC2 생성 과정

1. **AMI 선택**
    - Amazon Linux, Ubuntu, Windows Server, CentOS
2. **인스턴스 타입 선택**
    - CPU, 메모리, 네트워크 성능 결정
3. **네트워크 설정**
    - VPC, 서브넷, 퍼블릭 IP 여부
4. **보안 그룹 설정**
    - SSH(22), HTTP(80), HTTPS(443) 등 포트 설정
5. **키 페어 생성**
    - 비밀번호 대신 사용, .pem 파일로 다운로드

---

## 🔑 EC2 접속 방법

### Linux/Mac

```bash
chmod 400 my-key.pem
ssh -i my-key.pem ec2-user@퍼블릭IP주소
```

### Windows

1. AWS 콘솔 → EC2 → 인스턴스 선택 → “연결”
2. PuTTY 사용 → .pem → .ppk 변환 후 접속

## 💰 EC2 요금 구조

| 요금 유형 | 특징 | 장점/단점 |
| --- | --- | --- |
| 온디맨드 | 시간당 고정 요금, 약정 없음 | 유연하지만 비용 높음 |
| 예약 인스턴스 | 1~3년 약정, 최대 75% 할인 | 장기간 사용 시 유리 |
| 스팟 인스턴스 | 경매 방식, 최대 90% 할인 | 중단 가능성 있음, 실험/배치용 적합 |

## 🔒 보안 베스트 프랙티스

1. **보안 그룹**
    - 필요한 포트만 열기, 신뢰 IP만 허용
    - 0.0.0.0/0 허용 금지
2. **키 관리**
    - 키 안전하게 보관, 권한 400 설정
    - 공개 저장소 업로드 금지
3. **정기 업데이트**

```bash
sudo yum update -y
```

## 📊 핵심 요약 표

| 개념 | 설명 |

### 💻 실습 내용 정리

- AMI 선택 후 인스턴스 생성
- t3.micro로 웹서버 배포
- 보안 그룹에서 SSH/HTTP 포트 설정
- 키 페어로 SSH 접속 실습
- 온디맨드 vs 예약 vs 스팟 인스턴스 요금 비교

### ❗ 헷갈렸던 점 / 문제 해결:

| 문제 | 해결 |
| --- | --- |
| 온디맨드/예약/스팟 요금 차이 | 각 요금 모델 특징 이해 후 적절히 선택 |
| 키 페어 분실 시 접속 문제 | 키 파일 안전하게 백업 필요 |
| 포트 설정 실수 | 보안 그룹에서 허용 포트 재확인 |

### 💡 느낀 점 / 배운 점:

- EC2는 서버 환경을 유연하게 만들 수 있는 핵심 서비스
- 인스턴스 타입과 요금 모델 이해 중요
- 키 관리 및 보안 그룹 설정이 서버 보안 핵심
