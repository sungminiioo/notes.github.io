## 📝 핵심 개념 요약

- **제네릭(Generics)**: 타입을 변수처럼 만들어 재사용성을 극대화하는 기능
- **인덱스드 엑세스 타입**: `T['key']` 형태로 타입 안의 특정 속성 타입만 뽑기
- **keyof**: 객체 타입의 키들을 유니온 타입으로 가져오기
- **맵드 타입**: 객체의 모든 key를 순회하며 새로운 타입 만들기
- **템플릿 리터럴 타입**: 문자열 패턴을 타입으로 정의
- **조건부 타입**: 타입 레벨에서 삼항 연산
- **infer**: 조건부 타입에서 타입 추론
- **유틸리티 타입**: TypeScript 기본 제공 타입 변환 모음(Partial, Pick 등)

## 📊 핵심 요약 표

| 개념 | 설명 | 예시 |
| --- | --- | --- |
| **제네릭** | 타입을 변수처럼 사용 | `<T>` |
| **Indexed Access Type** | 특정 속성의 타입 추출 | `User['id']` |
| **keyof** | 모든 키를 유니온으로 | `"id" |
| **Mapped Type** | 속성 전체 변환 | `{ [K in keyof T]: ... }` |
| **조건부 타입** | 조건문 기반 타입 | `T extends U ? A : B` |
| **infer** | 타입 추론 | `infer R` |
| **Utility Types** | 미리 만들어진 타입 변환 | `Partial<T>` |

### 💡 느낀 점 / 배운 점:

- 제네릭은 단순 “타입의 빈칸”으로 생각하면 이해가 빨라짐
- Mapped Type과 keyof 조합이 실무에서 **타입 자동화**에 매우 강력함
- 조건부 타입 + infer 조합으로 타입 레벨 로직까지 짤 수 있음
- 유틸리티 타입(Partial, Pick 등)은 사실 조건부/맵드 타입 기반이라 원리를 알면 응용 쉬움
