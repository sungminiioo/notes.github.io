## 📝 핵심 개념 요약

- CloudWatch는 AWS 리소스의 실시간 **모니터링·로그 수집·알람·시각화** 서비스
- CPU·메모리·네트워크 등 **지표(메트릭)** 확인 가능
- 특정 조건 이상 시 **알람** 발송
- 로그를 중앙에서 관리 가능
- 대시보드로 메트릭을 시각적으로 통합
- 비용·지연·보존기간 등 고려 필요
- 실습: **EC2 CPU 사용률 알람 생성**, IAM 권한 문제 해결, SNS 주제 생성

### 💻 실습 내용 정리

### **1️⃣ CloudWatch에서 CPU 경보 생성 절차**

- CloudWatch 서비스 이동
- 좌측 "경보" → "경보 생성" 클릭
- 지표 선택: `EC2 → Per-Instance Metrics → CPUUtilization`
- 임계값 70 설정
- 알림 작업에서 SNS 주제 생성 시도 → **IAM 권한 부족으로 실패**

### **2️⃣ IAM 권한 문제 해결**

- IAM → 사용자 → 현재 계정 선택
- "권한 추가" → 직접 정책 연결
- `AmazonSNSFullAccess` 추가
- 다시 CloudWatch로 돌아와 SNS 주제 생성 성공

### **3️⃣ 알람 최종 생성**

- 주제 이름 입력 (예: cpu-alarm-topic)
- 알림 이메일 작성
- 알람 이름·설명 작성
- 미리보기 확인 후 ‘경보 생성’

### ❗ 헷갈렸던 점 / 문제 해결:

- **SNS 주제 생성 실패 원인?**
    
    → IAM 계정에 SNS 생성 권한 부족
    
- **해결 방법**
    
    → `AmazonSNSFullAccess` 정책 직접 연결
    
- **CPU 임계값 단위 입력**
    
    → `%` 없이 숫자(70)만 입력해야 함
    

### 💡 느낀 점 / 배운 점:

- CloudWatch는 단순 모니터링이 아니라 “장애 발생 전 예방 도구”라는 것을 이해함
- IAM 권한이 실제 실무에서 얼마나 중요한지 다시 체감
- 알람 시스템 구축이 필수적인 운영 업무임을 알게 됨
