## 📝 핵심 개념 요약

### 1. EC2 인스턴스 생성 과정

1. **리전 선택**: 서울(ap-northeast-2)
2. **AMI 선택**: Amazon Linux 2023 (프리티어 가능 여부 확인)
3. **인스턴스 타입 선택**: t3.micro (프리티어)
4. **키페어 생성**: SSH 접속용, `.pem` 다운로드
5. **네트워크/보안 그룹 설정**: SSH 허용, 위치 제한 필요 시 설정
6. **스토리지 구성**: 기본값 8GB

### 2. EC2 접속 방법

- **AWS 콘솔 내 연결**: “연결” 버튼 클릭 → 터미널 접속
- **SSH 클라이언트 연결**:
    
    ```bash
    chmod 400 "키파일.pem"     # MAC/Linux
    ssh -i "키파일.pem" ec2-user@퍼블릭IP
    ```
    
- Windows는 .pem → .ppk 변환 후 PuTTY 사용
- 연결 종료: `exit` 또는 `Ctrl+D`

### ❗ 헷갈렸던 점 / 문제 해결:

| 문제 | 해결 |
| --- | --- |
| 키 권한 오류 | MAC/Linux: `chmod 400`, Windows: 보안 주체 조정 |

- 프리티어 제한을 초과하면 요금 발생 → 관리 필요
