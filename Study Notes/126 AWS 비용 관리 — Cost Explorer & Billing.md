### 📘 오늘 공부한 주제:

> AWS 비용 확인, 프리티어 사용량 점검, 예산(Budget) 설정
> 

---

## 📝 핵심 개념 요약

- *Billing(결제)**에서 서비스별 요금 및 예상 청구 비용 확인 가능
- **Free Tier 사용량**을 MTD(이번 달 누적) 기준으로 확인 가능
- *예산(Budget)**을 설정하면 비용 초과 시 이메일 알림을 받을 수 있음
- 잘못 구성된 리소스로 인해 **숨은 비용이 발생하는 것을 예방**하는 핵심 기능

## 📊 핵심 요약 표

| 항목 | 요약 |
| --- | --- |
| Billing | 현재 월 예상 비용 & 서비스별 비용 확인 |
| Free Tier | 프리 티어 내 사용량 관리 및 초과 여부 체크 |
| Budget | 비용 초과 시 이메일 경고 알림 |
| 사용 목적 | 과금 위험 방지 & 관리 효율화 |
| 주의점 | root 계정으로만 Billing 조회/예산 생성 가능 |

### 💻 실습 내용 정리

### 1) Billing 메뉴 이동

- root 계정 로그인
- 콘솔 오른쪽 상단 → **결제 및 비용 관리(Billing)** 클릭

### 2) 서비스별 요금 확인

- 좌측 메뉴 → **청구서(Bill)**
- 활성화된 서비스들의 **현재 사용 요금** 확인 가능

### 3) Free Tier 사용량 확인

- 좌측 메뉴 → **프리 티어**
- MTD 사용량 & 예상 사용량 확인 → 초과 가능성 미리 체크

### 4) 예산 생성

- 좌측 메뉴 → **예산(Budgets)** → 예산 생성
- 프리 티어 계정에서는 **최대 2개까지 생성 가능**
- 이메일 수신자 등록(쉼표로 여러 명 설정 가능)

### ❗ 헷갈렸던 점 / 문제 해결:

| 헷갈렸던 점 | 해결 방법 |
| --- | --- |
| 청구 메뉴가 보이지 않는 문제 | root 계정이 아니면 Billing 메뉴 제한됨 → root 계정으로 로그인 |
| 프리 티어 사용량이 갑자기 증가하는 이유 | EC2·S3·ALB 등 실수로 켜둔 리소스 확인 필요 |
| 예산 알림이 오지 않는 문제 | Budget 생성 후 이메일 인증 여부 확인 |

### 💡 느낀 점 / 배운 점:

- AWS 비용 관리는 단순히 보는 것이 아니라 **예산 설정 & 프리 티어 모니터링**이 필수라는 걸 깨달음
- Billing 콘솔에서 작은 비용 증가만으로도 사용 중인 리소스를 빠르게 파악할 수 있어
    
    **불필요한 과금 방지에 매우 효과적**
    
- 개발 환경이라고 방치하면 금액이 쌓일 수 있기 때문에
    
    AWS 사용 시 비용 감각을 함께 갖는 것이 중요하다고 느낌
