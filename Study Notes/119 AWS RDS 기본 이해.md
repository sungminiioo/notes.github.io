## 📝 핵심 개념 요약

### 1. RDS란?

| 개념 | 설명 |
| --- | --- |
| **RDS (Relational Database Service)** | AWS에서 제공하는 관리형 관계형 데이터베이스 서비스 |
| **비유** | S3: 창고, RDS: 도서관 카드 목록처럼 정리된 표 형태 데이터 저장 |
| **용도** | 정형 데이터 저장 및 관리, 쿼리 수행 |

### 2. RDS 필요성

| 기존 방식 | 문제점 |
| --- | --- |
| 직접 서버에 DB 설치 | 서버 관리, 백업, 보안 패치 직접 수행, 모니터링 필요, 성능 최적화 어려움 |
| RDS 사용 | AWS가 백업·패치·모니터링 대신 수행, 클릭 몇 번으로 확장 가능, 장애 시 자동 복구 |

### 3. RDS vs 일반 DB

| 구분 | 일반 DB 서버 | AWS RDS |
| --- | --- | --- |
| 설치 | 직접 설치 및 설정 | 클릭 몇 번으로 생성 |
| 백업 | 수동 스크립트 | 자동 백업 (최대 35일) |
| 보안 패치 | 직접 적용 | 자동 적용 |
| 모니터링 | 별도 도구 필요 | CloudWatch로 자동 모니터링 |
| 확장 | 서버 증설 필요 | 클릭 몇 번으로 확장 |

### 4. 지원 데이터베이스 엔진

- MySQL, PostgreSQL, MariaDB, Oracle, SQL Server, Aurora

### 5. 핵심 특징

| 특징 | 설명 |
| --- | --- |
| 자동 백업 | 매일 자동 백업, Point-in-time 복구 가능 |
| 다중 가용 영역 (Multi-AZ) | 메인 DB 장애 시 자동 전환, 거의 무중단 |
| 읽기 전용 복제본 | 읽기 작업 분산, 메인 DB 성능 향상 |

### 6. S3 vs RDS 비교

| 항목 | S3 | RDS |
| --- | --- | --- |
| 용도 | 이미지, 동영상, 문서 | 정형 데이터 저장/조회 |
| 데이터 형태 | 비정형 | 정형 |
| 예시 | 프로필 사진, 동영상 | 회원 정보, 주문 내역, 재고 관리 |

### 7. 요금 구조

| 비용 요소 | 설명 |
| --- | --- |
| 인스턴스 크기 | CPU, 메모리 사양 |
| 스토리지 | 저장 공간 |
| 네트워크 | AWS 서비스 간 데이터 전송 |
| 백업 | 무료 용량 초과 시 비용 발생 |

💡 **비용 최적화 팁**

- 개발/테스트 환경: 작은 인스턴스 사용
- 프리티어 활용: 12개월 무료
- 예약 인스턴스: 장기 할인 적용

- RDS는 직접 DB 서버를 관리하는 부담을 크게 줄여줌
- 자동 백업, Multi-AZ, Read Replica 기능으로 안정성·성능 향상
- AWS 서비스 간 S3와 RDS의 용도 차이를 명확히 이해
