## 📝 핵심 개념 요약

### 1. 정적 웹사이트 호스팅

| 개념 | 설명 |
| --- | --- |
| **정적 웹사이트 호스팅** | S3 버킷을 웹 서버처럼 사용하여 HTML, CSS, JS 파일을 인터넷에 공개 |
| **특징** | 서버 관리 불필요, 비용 저렴, 높은 가용성 및 확장성 |
| **웹사이트 엔드포인트** | S3 버킷 속성에서 확인 가능, 퍼블릭 URL 제공 |

### 2. 버킷 정책

| 개념 | 설명 |
| --- | --- |
| **버킷 정책** | JSON 형식으로 S3 접근 권한 제어 |
| **퍼블릭 접근 허용** | Principal: `*`, Action: `s3:GetObject`, Resource: `버킷ARN/*` |
| **보안 유의점** | 민감한 파일은 퍼블릭 접근 금지, 필요한 파일만 공개 |

## 📊 핵심 요약 표

| 웹사이트 엔드포인트 | S3 버킷 URL, 퍼블릭 접근 시 웹에서 접속 가능 |

### 💻 실습 내용 정리

| 단계 | 내용 |
| --- | --- |
| 1 | AWS 콘솔에서 S3 서비스 접속, 서울 리전 확인, 버킷 생성 |
| 2 | 퍼블릭 엑세스 체크 해제하여 외부 접근 가능 설정 |
| 3 | 생성된 버킷 → 속성 → 정적 웹사이트 호스팅 활성화, 인덱스 문서(`index.html`) 지정 |
| 4 | 권한 → 버킷 정책 편집 → 정책 생성기 사용, GetObject 권한 부여 |
| 5 | 생성된 정책(JSON) 붙여넣기 후 저장 |
| 6 | `index.html` 파일 업로드 |
| 7 | 버킷 웹사이트 엔드포인트 URL 클릭 → 웹사이트 접속 확인 |

### ❗ 헷갈렸던 점 / 문제 해결:

| 문제 | 해결 |
| --- | --- |
| S3 웹사이트 URL 접속 시 AccessDenied | 버킷 정책에서 `s3:GetObject` 권한과 ARN 뒤에 `/*` 붙였는지 확인 |
| 인덱스 문서 지정 문제 | 정적 웹사이트 호스팅 설정에서 `index.html` 정확히 입력 후 저장 |
| 퍼블릭 접근 오류 | 퍼블릭 액세스 차단 해제 및 정책 설정 확인 |

### 💡 느낀 점 / 배운 점:

- S3로 서버 없이 간단하게 정적 웹사이트 운영 가능
- 정책 설정과 퍼블릭 액세스 관리가 필수, 잘못 설정하면 접근 불가
- index.html 업로드와 엔드포인트 확인 과정이 직관적
- 실습을 통해 AWS S3 웹사이트 호스팅 전체 과정 이해
